## AIコーディングエージェントの本当の攻撃面は設定ファイルだった

https://zenn.dev/ju571n/articles/ai-agent-config-attack-surface

AIコーディングエージェントの脆弱性はモデルの挙動ではなく、設定ファイルによる権限管理の不備にあり、これらを監視するセキュリティツール「Sigil」が解決策として提示されています。

AIコーディングエージェント（Claude Code, Cursor, GitHub Copilot等）における深刻なセキュリティリスクの起点は、AIモデルそのものではなく「設定ファイル」にあることを詳説しています。TrustFallやAWS Kiroの事例では、リポジトリ内の`.claude/settings.json`や`.mcp.json`が悪用され、クローンして開くだけでRCE（リモートコード実行）が可能になる、あるいは間接プロンプトインジェクションにより権限設定が書き換えられる脆弱性が報告されています。これらの脆弱性（CVE-2025-59536等）は、信頼確認ダイアログの前に設定が読み込まれる仕様を突いたものです。これに対処するため、筆者は設定ファイルを監視し危険度をスコアリングするRust製ツール「Sigil」を公開しました。Sigilは設定の変更をリアルタイムで検知し、SIEM等へログを流すことで、開発者の利便性を損なわずにAI-SPM（AI Security Posture Management）を実現します。
