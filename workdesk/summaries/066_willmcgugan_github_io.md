## ターミナルでのAI体験を統合する「Toad」：複数エージェントを一つの洗練されたTUIで操作

https://willmcgugan.github.io/toad-released/

**Original Title**: Toad is a unified experience for AI in the terminal

Pythonの「Rich」や「Textual」ライブラリの作者として知られるWill McGugan氏が、ターミナル上で動作するAIエージェントのフロントエンド「Toad」をリリースした。現在、AIコーディングツールはClaude CodeやGemini CLI、OpenHandsなど多岐にわたり、それぞれが独自のCLIインターフェースを持っている。著者はこの断片化された現状に対し、これら複数のエージェントを一つの洗練されたTUI（Terminal User Interface）の下でシームレスに操作できる環境が必要だと考え、Toadを開発した。

Toadの最大の特徴は、ACP（Agent Control Protocol）プロトコルを採用することで、現時点で12種類以上のAIエージェントCLIをサポートしている点だ。ユーザーは使い慣れたターミナルを離れることなく、単一のUIから異なるAIモデルやツールを切り替えて利用できる。著者は、単なる「CLIのラッパー」に留まらない、GUIに匹敵するリッチな体験をターミナルで実現することに注力している。例えば、ファイルコンテキストを導入する際の「@」メンションによる高速なファジー検索機能は、.gitignoreを考慮した設計となっており、開発者が期待する直感的な操作感を提供する。

プロンプトエディタの完成度も極めて高い。ターミナル上でありながら、マウスやキーボードによる選択、コピー＆ペーストが可能で、入力中にはMarkdownのシンタックスハイライトがリアルタイムで適用される。また、Markdownのストリーミング描画についても、著者が培ってきた技術を駆使し、大規模なドキュメントでも表やコードフェンスを含めて高速かつ正確にレンダリングする。

さらに実用的な側面として、フル機能のシェル統合が挙げられる。従来のAIツールの多くはコマンドの出力を表示するだけだったが、Toadは対話的なTUIアプリ（別のターミナルアプリなど）もカラーやマウス操作を維持したまま実行できる。「!」から始まるコマンド入力や、シェルのタブ補完機能、さらにはJupyter Notebookのように過去の会話ブロックを遡って再利用できる機能も備えている。著者は、AIとの対話が従来のターミナルベースのワークフローの自然な延長線上にあるべきだと主張しており、Toadはそのビジョンを具現化したツールとなっている。
