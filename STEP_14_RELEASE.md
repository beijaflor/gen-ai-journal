### 2. Verify Past Release Format

Before creating a new release, check the format of recent releases:

```bash
# Using GitHub API
curl -s https://api.github.com/repos/beijaflor/gen-ai-journal/releases | head -200

# Extract release body from recent releases
curl -s https://api.github.com/repos/beijaflor/gen-ai-journal/releases | jq -r '.[0].body'
```

### 3. Create GitHub Release

#### Option A: Via GitHub Web UI (Recommended)

1. Navigate to: https://github.com/beijaflor/gen-ai-journal/releases/new
2. Select tag: `YYYY-MM-DD` (from dropdown)
3. Release title: `YYYY-MM-DD`
4. Copy content from `release-description-YYYY-MM-DD.md` into description
5. Click "Publish release"

#### Option B: Via GitHub CLI (if available)

```bash
# Create release from tag with description file
gh release create YYYY-MM-DD \
  --title "YYYY-MM-DD" \
  --notes-file release-description-YYYY-MM-DD.md
```

#### Option C: Via GitHub API

```bash
# Create release using curl
curl -X POST \
  -H "Accept: application/vnd.github+json" \
  -H "Authorization: token YOUR_TOKEN" \
  https://api.github.com/repos/beijaflor/gen-ai-journal/releases \
  -d '{
    "tag_name": "YYYY-MM-DD",
    "name": "YYYY-MM-DD",
    "body": "'"$(cat release-description-YYYY-MM-DD.md)"'",
    "draft": false,
    "prerelease": false
  }'
```

## Release Description Format Details

### Website Section
Links to the published journal on GitHub Pages:
- **Main Journal**: `/journals/YYYY-MM-DD/main/` - Curated high-impact articles
- **Annex Journal**: `/journals/YYYY-MM-DD/annex/` - Unique perspective articles (B-side)
- **Summaries**: `/journals/YYYY-MM-DD/summaries/` - Complete list of all summaries

### Generated by Claude Section
Links to source markdown files in the repository:
- **00_weekly_journal_**: Main journal source file
- **01_annex_journal_**: Annex journal source file
- **02_omitted_summaries**: Summaries not included in main/annex journals

**Important**: All links use the tag name as the branch reference (e.g., `blob/YYYY-MM-DD/`) to ensure they point to the exact version at release time.

## Common Issues

### Issue 1: Links Return 404
**Cause**: GitHub Pages may take a few minutes to deploy after merge
**Solution**: Wait 2-5 minutes and try again

### Issue 2: Source File Links Point to Wrong Version
**Cause**: Using `main` branch instead of tag name in URLs
**Solution**: Always use `blob/YYYY-MM-DD/` not `blob/main/`

### Issue 3: Day of Week is Wrong
**Cause**: Incorrect date conversion
**Solution**: Use `date` command or verify manually with calendar
